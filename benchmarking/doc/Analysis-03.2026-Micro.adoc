== Executive Summary

JMH micro-benchmark testing of the core JWT validation library:

* **Core Validation**: 159,100 ops/s, P50=77µs
* **Signature validation** dominates at 68µs (88% of total)
* **Fast-fail on invalid tokens**: 222,300 ops/s (+39%)
* **Integration gap**: ~6x throughput difference vs WRK at 50 connections — explained by link:Analysis-03.2026-Latency-Decomposition.adoc[ablation study]

== Configuration

* **Tool**: JMH 1.37 (Java Microbenchmark Harness)
* **JVM**: OpenJDK 21.0.7 (Temurin), aarch64 (Apple M4)
* **Threads**: 100 concurrent
* **Iterations**: 5 measurement + 3 warmup, 4 seconds each
* **Mode**: Throughput (ops/s) and Average Time (µs/op)

The ~6x throughput gap and ~30x latency gap are fully explained by integration overhead. The link:Analysis-03.2026-Latency-Decomposition.adoc[ablation study] decomposes the 2.30ms into shared infrastructure (33.9%), JAX-RS routing (17.4%), and JWT validation under concurrency (46.1%).

== Conclusion

The core library is highly optimized at 77µs P50. The integration overhead is expected for HTTP-based microservices. Signature validation (RS256) dominates at 88% — this is the primary candidate for optimization, but no alternative has been benchmarked.
