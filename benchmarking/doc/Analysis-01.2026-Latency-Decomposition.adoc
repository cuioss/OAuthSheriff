== Overview

A **mock JWT endpoint** (`/mock-jwt/validate`) was created to directly measure the latency decomposition. The mock endpoint replicates the JWT endpoint's full HTTP processing path ‚Äî CDI injection, `HttpServletRequestResolver.resolveHeaderMap()`, `Authorization` header extraction, and `ValidationResponse` JSON serialization ‚Äî but **skips the actual JWT library validation** (`tokenValidator.createAccessToken()`). This turns the inferred decomposition into a measured one.

An **ablation study** further decomposes the gap between the health endpoint and mock JWT endpoint by progressively adding processing layers. Four additional endpoints isolate individual cost components:

=== Key Findings

**JAX-RS POST routing is the dominant mock-to-health gap component.** The 0.381ms cost of JAX-RS POST routing + `@RunOnVirtualThread` dispatch + Jackson serialization accounts for 92% of the 0.414ms gap between the health and mock JWT endpoints. This is expected ‚Äî POST requests with JSON serialization of a `ValidationResponse` record require more processing than the SmallRye Health GET endpoint.

**CDI `@ApplicationScoped` injection has zero per-request cost.** Variant A (zero CDI, 0.870ms in original run) was actually _slower_ than variant B (CDI-injected, 0.784ms in original run). Since CDI overhead would make B slower, this confirms `@ApplicationScoped` beans are resolved at startup, not per request.

**Vert.x context resolution is minimal.** The `resolveHeaderMap()` call adds only 0.023ms (23¬µs), a small cost for accessing the Vert.x request context and extracting HTTP headers into a `Map<String, List<String>>`.

**Authorization parsing is negligible.** The full header parsing pipeline ‚Äî extracting the `authorization` key, checking the `Bearer ` prefix, substring extraction, and `HashMap` construction ‚Äî costs only 0.010ms (10¬µs), effectively free.

**JWT validation under concurrency is the dominant cost at 55.8% (1,160¬µs) ‚Äî measured, not estimated.** The Direct Validation variant (D) calls `createAccessToken()` directly, bypassing CDI `Instance.get()` and `@Timed` instrumentation. The 1.160ms gap between D and Mock JWT isolates the pure JWT validation cost under 50 concurrent connections ‚Äî 15x slower than the 77¬µs JMH baseline due to cache contention, memory barriers, and real object allocation under concurrency.

**CDI `Instance.get()` + `@Timed` overhead is only 50¬µs (2.4%) ‚Äî far lower than the previous estimate of 15-30%.** The 0.050ms gap between the full JWT endpoint and variant D measures the actual cost of CDI proxy resolution (`CreationalContext`, `InjectionPoint` introspection) plus Micrometer `@Timed` instrumentation. This is negligible compared to the JWT validation itself.

=== Ablation Variant Descriptions

* **A: Bare Baseline** (`/mock-jwt-bare/baseline`) ‚Äî Was a separate JAX-RS class with zero CDI constructor injection, removed after benchmarking. Returned a pre-built `ValidationResponse`. Isolated JAX-RS + `@RunOnVirtualThread` + Jackson without any CDI beans. Benchmark data is from the original run.
* **B: JAX-RS Baseline** (`/mock-jwt/baseline`) ‚Äî Same behavior as A, but in the CDI-injected `MockJwtValidationEndpoint` class. Comparing A vs B confirms that `@ApplicationScoped` constructor injection has zero per-request cost.
* **C: Header Only** (`/mock-jwt/header-only`) ‚Äî Calls `resolveHeaderMap()` but does not parse headers or construct a HashMap. Returns a pre-built response. The difference C minus B isolates the Vert.x context resolution cost.
* **D: Direct Validation** (`/mock-jwt/direct-validate`) ‚Äî Calls `createAccessToken()` directly on the injected `TokenValidator`, bypassing CDI `Instance.get()` proxy resolution and `@Timed` instrumentation. The difference D minus Mock JWT isolates the pure JWT validation cost under concurrency, while JWT minus D isolates the CDI overhead.

**Key P99 observations:**

* **JWT validation dominates tail latency.** The P99 jump from Mock JWT (8.98ms) to Direct Validation (18.44ms) is 9.46ms ‚Äî the single largest P99 delta, confirming that `createAccessToken()` under contention is the primary source of tail latency.
* **CDI overhead _reduces_ P99.** The full JWT endpoint (14.30ms P99) has a _lower_ P99 than Direct Validation (18.44ms). This suggests that CDI `Instance.get()` or `@Timed` may introduce a slight serialization effect that reduces extreme outliers, or this is simply run-to-run variance at P99.
* **P99/P50 ratios compress as processing increases.** Health has a 16x P99/P50 ratio (dominated by infrastructure jitter), while the full JWT endpoint has only 6.9x ‚Äî heavier per-request work reduces the relative impact of infrastructure variance.

== JMH vs Integration Validation Cost

The JMH micro-benchmark measures isolated JWT library validation at 0.077ms (77¬µs). The ablation study reveals the actual JWT validation cost under concurrency is 1.160ms (variant D minus Mock JWT) ‚Äî **15.1x more** than the isolated library. This gap is now fully decomposed into two independently measured factors:

**Primary factor (55.8%, measured): JWT validation under concurrency = 1,160¬µs**

* **`tokenValidator.createAccessToken()` under concurrency** ‚Äî 50 concurrent connections competing for the lock-free cache, CPU cache line invalidation, memory barriers
* **`AccessTokenContent` construction** ‚Äî real object allocation from parsed claims (vs JMH reusing pre-parsed data)
* 15.1x concurrency multiplier over the 77¬µs JMH baseline

**Secondary factor (2.4%, measured): CDI `Instance.get()` + `@Timed` = 50¬µs**

* **CDI `Instance<BearerTokenResult>` proxy resolution** ‚Äî CDI creates `CreationalContext` and invokes the producer per request
* **`InjectionPoint` metadata extraction** ‚Äî CDI inspects `@BearerToken` annotation parameters
* **`@Timed` micrometer instrumentation** ‚Äî metrics recording overhead on every validation
* At 50¬µs, this is far lower than the earlier JFR-based estimate of 160-325¬µs

The previous (inferred) decomposition attributed 74% to "CDI/serialization" and only 4% to the JWT library. The measured data reverses this: header extraction and serialization cost only 20% (0.414ms across three sub-layers), while JWT validation under concurrency alone accounts for 55.8% (1.160ms) and CDI overhead is a negligible 2.4% (0.050ms).
