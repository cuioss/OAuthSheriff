== Purpose

JWT validation performance characteristics, baselines, and analysis for the CUI OAuth Sheriff Quarkus integration.

== Performance Baselines (March 2026)

=== System Configuration

* **Platform**: Apple M4, 10 CPU cores, 32 GB RAM
* **Runtime**: Mandrel jdk-25 native image
* **Framework**: Quarkus 3.27.2 (LTS)
* **Container**: Docker with Keycloak for token issuance

=== Current Performance Summary

[cols="3,2,2,2", options="header"]
|===
| Endpoint | Throughput | Avg Latency | P99 Latency

| **JWT Validation** | 25,178 ops/s | 2.30ms | 11.52ms
| **Health Check** | 98,211 ops/s | 0.78ms | 6.04ms
| **JWT Library (JMH)** | 159,100 ops/s | 77µs (P50) | 163µs
|===

All measurements at 50 concurrent connections (optimal). See link:../../../benchmarking/doc/Analysis-03.2026-Integration.adoc[Integration Analysis] for connection sweep (50-300).

== Latency Decomposition

From the link:../../../benchmarking/doc/Analysis-03.2026-Latency-Decomposition.adoc[ablation study]:

[cols="3,2,2", options="header"]
|===
| Layer | Latency (avg) | % of total

| Shared infrastructure (Docker, TLS, HTTP/2, Quarkus routing) | 0.78ms | 33.9%
| JAX-RS POST routing + Jackson serialization | 0.40ms | 17.4%
| Vert.x context resolution | 0.06ms | 2.6%
| Authorization parsing + HashMap construction | ~0ms | ~0%
| **JWT validation under concurrency** | **1.06ms** | **46.1%**
| CDI proxy + `@Timed` | ~0ms | ~0%
|===

**Total**: 0.78 + 0.40 + 0.06 + 0 + 1.06 + ~0 ≈ **2.30ms**

JWT validation under concurrency is the sole significant cost. The core library validates in 77µs isolation (JMH); under 50 concurrent connections the measured cost is 1,060µs — a ~14x increase. The exact causes of this concurrency multiplier are not individually measured. See link:../../../benchmarking/doc/Analysis-03.2026-Integration.adoc#_root_cause_jwt_validation_under_concurrency[root cause analysis].

== Algorithm Performance

[cols="2,2,2,3", options="header"]
|===
| Algorithm | Type | Security Level | Notes

| RS256/384/512 | RSA | 2048-bit | 68µs — current default
| ES256/384/512 | ECDSA | 256-384-bit curves | Not benchmarked
| Ed25519/Ed448 | EdDSA | Curve25519/448 | Not benchmarked
|===

No alternative algorithm has been benchmarked. Algorithm choice should be driven by standards compliance, token size, and key management.

== Benchmark Execution

[source,bash]
----
# Integration benchmarks
cd oauth-sheriff-quarkus-parent/oauth-sheriff-quarkus-integration-tests
../../mvnw clean verify -Pintegration-tests

# JMH microbenchmarks (run from repository root)
cd benchmarking/benchmark-core
../../mvnw clean verify -Pbenchmark

# JFR profiling
docker compose -f docker-compose.yml -f docker-compose.jfr.yml up -d
----

**Regression Thresholds**: >10% throughput loss, >20% latency increase, >50% memory increase.
