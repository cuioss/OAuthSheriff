== Overview

The oauth-sheriff-quarkus extension automatically exposes metrics when the Quarkus Micrometer extension is present. The `JwtMetricsCollector` registers Micrometer counters and timers, using scheduled polling (every 10 seconds) to update from `SecurityEventCounter` and `TokenValidatorMonitor`.

== Available Metrics

[cols="2,1,3,2"]
|===
|Metric Name |Type |Description |Tags

|`sheriff.oauth.validation.errors` |Counter |JWT validation errors by type |event_type, result, category
|`sheriff.oauth.validation.duration` |Timer |Validation pipeline step durations |step
|===

=== Error Metric Tags

* **`event_type`**: e.g., `TOKEN_EXPIRED`, `SIGNATURE_VALIDATION_FAILED`
* **`result`**: Always "failure"
* **`category`**: `STRUCTURE`, `SIGNATURE`, or `SEMANTIC`

=== Performance Metric Steps

`complete_validation`, `token_parsing`, `header_validation`, `signature_validation`, `claims_validation`, `jwks_operations`

=== Supported Event Types

* **Structure**: `TOKEN_EMPTY`, `TOKEN_SIZE_EXCEEDED`, `INVALID_JWT_FORMAT`, `FAILED_TO_DECODE_JWT`, `FAILED_TO_DECODE_HEADER`, `FAILED_TO_DECODE_PAYLOAD`, `DECODED_PART_SIZE_EXCEEDED`
* **Semantic**: `MISSING_CLAIM`, `MISSING_RECOMMENDED_ELEMENT`, `TOKEN_EXPIRED`, `TOKEN_NBF_FUTURE`, `AUDIENCE_MISMATCH`, `AZP_MISMATCH`, `ISSUER_MISMATCH`
* **Signature**: `NO_ISSUER_CONFIG`, `SIGNATURE_VALIDATION_FAILED`, `KEY_NOT_FOUND`, `UNSUPPORTED_ALGORITHM`
* **JWKS**: `JWKS_FETCH_FAILED`, `JWKS_JSON_PARSE_FAILED`, `FAILED_TO_READ_JWKS_FILE`, `KEY_ROTATION_DETECTED`
* **Token Creation**: `ACCESS_TOKEN_CREATED`, `ID_TOKEN_CREATED`, `REFRESH_TOKEN_CREATED`

== Configuration

Enabled automatically when `quarkus-micrometer` is present. Add to your project:

[source,xml]
----
<dependency>
    <groupId>io.quarkus</groupId>
    <artifactId>quarkus-micrometer</artifactId>
</dependency>
<!-- Optional: Prometheus registry -->
<dependency>
    <groupId>io.quarkus</groupId>
    <artifactId>quarkus-micrometer-registry-prometheus</artifactId>
</dependency>
----

Metrics available at `/q/metrics`.

== Monitoring Examples

=== Prometheus Queries

[source,promql]
----
# Total JWT validation errors
sum(sheriff_oauth_validation_errors_total)

# Error rate by category
rate(sheriff_oauth_validation_errors_total[5m]) by (category)

# Signature verification failures
sum(sheriff_oauth_validation_errors_total{event_type="SIGNATURE_VALIDATION_FAILED"})

# Average signature validation duration
rate(sheriff_oauth_validation_duration_seconds_sum{step="signature_validation"}[5m]) / rate(sheriff_oauth_validation_duration_seconds_count{step="signature_validation"}[5m])

# 95th percentile of complete validation
histogram_quantile(0.95, rate(sheriff_oauth_validation_duration_seconds_bucket{step="complete_validation"}[5m]))
----

=== Alerting Examples

[source,yaml]
----
- alert: JwtSignatureVerificationFailures
  expr: rate(sheriff_oauth_validation_errors_total{event_type="SIGNATURE_VALIDATION_FAILED"}[5m]) > 0.1
  for: 5m
  labels:
    severity: critical
  annotations:
    summary: "JWT signature verification failures detected"

- alert: JwtSlowSignatureValidation
  expr: histogram_quantile(0.95, rate(sheriff_oauth_validation_duration_seconds_bucket{step="signature_validation"}[5m])) > 0.1
  for: 5m
  labels:
    severity: warning
  annotations:
    summary: "JWT signature validation P95 above 100ms"
----
